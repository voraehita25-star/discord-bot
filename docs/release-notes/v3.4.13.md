# v3.4.13 — Full-repo line-by-line audit (round 6): 2 high-severity fixes + 62 fixes across all stacks & dashboard rebuild

**Date:** 2026-06-17

Sixth pass in the v3.4.8–v3.4.12 line-by-line audit series. Every non-test source
file across all four stacks (Python, TypeScript, Rust, Go) plus the hand-authored
UI, scripts, SQL migrations and build/config was re-read **line-by-line** —
bin-packed into **77 work-units / 207 files / 93,910 lines**, each read by a
fan-out auditor and then **independently re-read by an adversarial,
refute-by-default verifier** that re-opens the cited code and confirms or rejects
each candidate. **122 agents** produced **62 confirmed findings** (**0 critical,
2 high, 15 medium, 37 low, 8 info**). All 62 were fixed; a small set of
opportunistic improvements was applied alongside. No features changed; no security
constraints were weakened (several were strengthened). Every suite stays green.

## High-severity fixes

- **Hot reload silently disabled for AI-core source under `cogs/ai_core/data/`**
  (`scripts/dev_watcher.py`). The ignore patterns `data/` / `data\` were
  normalized down to the bare token `data`, which then matched that path
  component **anywhere** in the tree — so the 7 tracked `.py` modules under
  `cogs/ai_core/data/` were watched and then silently dropped, giving no restart
  (and, outside debug mode, no warning) when a developer edited persona/data
  modules. Trailing-separator patterns are now anchored to the repo-root first
  path component (`rel.parts[0]`), so `data/` ignores `./data/` but not
  `cogs/ai_core/data/`.
- **`reindex_db` crashed on the success path after a durable commit**
  (`scripts/maintenance/reindex_db.py`). Assigning `conn.isolation_level = None`
  on the live `aiosqlite` connection runs the setter **on the caller thread**,
  tripping sqlite3's same-thread guard with `ProgrammingError` — *after* the
  destructive DROP/RENAME had already committed. The traceback printed right
  after `[DONE]`, `VACUUM` never ran, and the `sys.exit(2)` index-failure gate
  became unreachable (masking real index-recreation failures from CI). `VACUUM`
  now runs on a dedicated short-lived autocommit `sqlite3` connection after the
  aiosqlite connection closes; a missing-DB guard replaces the raw
  `FileNotFoundError`.

## Notable fixes by area (highlights from the 62)

- **Dashboard chat / persistence** — the Gemini backend now validates
  client-supplied `is_regeneration` against the DB (parity with the Claude
  backend), so a spoofed flag can no longer skip user-message persistence and
  document extraction; a documents-only message is no longer rejected as "Empty";
  the Claude frontend-history fallback guards non-dict items; the CLI backend no
  longer persists a phantom empty user row when every attachment fails to decode
  (`cogs/ai_core/api/`).
- **Reliability / resource** — the dashboard WebSocket server cleans up its
  `AppRunner` on every failed-start path (TLS-misconfig early-return and bind
  failure) instead of orphaning it; CLI temp-attachment cleanup suppresses
  `CancelledError` so a mid-cleanup cancellation can't leak files; the streaming
  Claude path records a circuit-breaker/failover **failure** on a 200-but-empty
  stream instead of reporting the sick endpoint healthy (`api_handler.py`,
  `ws_dashboard.py`).
- **Music** — per-guild volume/loop/24-7 settings are restored even when the DB
  queue is empty on restart (a 24/7 channel no longer silently drops `mode_247`);
  the `!fix` disconnect-failure path no longer skips the deferred `cleanup_pending`
  drain; the DB queue loader no longer reports success with a silently-emptied
  queue when every row is malformed; `format_duration` guards NaN/±inf
  (`cogs/music/`).
- **Memory / RAG** — the Python RAG fallback now enforces the same
  dimension/finite validation as the Rust backend on both `add` and search;
  `EntityFacts` coerces/guards non-dict `relationships`/`custom`; the large-history
  importance scan is offloaded off the event loop (`cogs/ai_core/memory/`).
- **Data preservation** — migration 002 now rebuilds `token_usage` /
  `conversation_summaries` with `INSERT…SELECT` (preserving rows on a legacy
  pre-migration DB) instead of `DROP`/recreate, guarded with `CREATE TABLE
  IF NOT EXISTS` so a fresh DB (where the source table doesn't exist yet) still
  migrates cleanly.
- **Go / scripts** — `url_fetcher`'s SSRF pre-check resolver now honors the
  request context/deadline; `install_all.ps1` no longer reports broken tools as
  `[OK]` (stdout was overriding `$?`) and degrades a transient download error to a
  per-tool failure; `build_rust.ps1` fails when an expected `.pyd` is missing;
  `_common.psm1` restores the dependency-gate default and cleans malformed PID
  files; `view_db.py`/`reindex_db.py` guard a missing database file.
- **Logging** — `StructuredFormatter` redacts secrets in individual string
  *leaves* before JSON serialization, so a keyword-prefixed secret in an
  `extra={…}` dict is removed **without** producing structurally-invalid JSON
  (`utils/monitoring/structured_logger.py`).

## Opportunistic improvements

- `scripts/dev_watcher.py`: `hashlib.md5` marked `usedforsecurity=False` (it is a

  High (B324) and documents intent.

  document persistence (Gemini backend) and the health-gated crash-retry counter
  reset (`dev_watcher`).
