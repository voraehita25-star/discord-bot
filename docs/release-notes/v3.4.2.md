# v3.4.2 — Whole-repo audit, feature trim & documentation sync

**Date:** 2026-06-02

Patch release built on a read-every-line audit of the **entire** source tree
(168 non-test files across Python / Rust / TypeScript / Go), an adversarially
verified bug-fix pass, removal of the dashboard's manual Long-term Memories
subsystem, a project-wide format, and a full documentation accuracy sweep.
Python suite **3,143 pass / 1 skip**; dashboard **190 vitest** + **70 Playwright**
+ strict `tsc` clean; `cargo check` clean on both Rust crates.

## Dashboard — Long-term Memories removed

The manually-curated **Long-term Memories** feature (the 🧠 dashboard page,
"+ Add Memory" UI, the global `dashboard_memories` SQLite table, the
`save/get/delete_memory` WebSocket handlers, and the `[Long-term Memories about
User]` system-prompt injection) has been **fully removed** across every layer —
backend, prompt pipeline (`build_user_context` is now a 2-tuple; the CLI prompt
builder dropped its `memories_context` param), frontend (`MemoryManager` class,
page, modal, nav, Ctrl shortcut), tests, and docs. Mapped with a multi-agent
sweep (92 references + 11 stragglers) so no dangling references remain.

> Unrelated and **kept**: per-conversation **document memory**
> (`dashboard_document_memories`, uploaded-file text), the Discord bot's
> auto-extracted `user_facts` / entity memory, and FAISS RAG.

> Note: existing databases keep the now-unused `dashboard_memories` table and
> its rows (orphaned, untouched — no destructive `DROP` was run).

## Correctness, security & reliability (22 verified fixes)

A full-project review found and an adversarial verifier confirmed 22 real
defects (1 high, 6 medium, 15 low). Highlights:

- **High — `scripts/bot_manager.py`**: `start_bot()` launched non-existent
  scripts (`start_bot.bat` / `start_dev_mode.bat` / `start_bot_hidden.vbs`), so
  every Start/Restart silently failed. Repointed to the real
  `scripts/startup/start.bat` / `dev.bat` (hidden mode now runs `start.ps1`
  hidden via PowerShell).
- **Security — `utils/monitoring/logger.py`**: a keyword-prefixed
  `Authorization: Bearer <jwt>` only redacted the JWT header and leaked the
  payload+signature. The keyword value class now consumes JWT dot-segments.
- **`cogs/ai_core/api/dashboard_handlers.py`** — `rename_conversation` /
  `save_memory` no longer crash the WebSocket connection on a non-string payload
  (coerce before `.strip()`); `dashboard_chat.py` does the same for `instruction`
  and guards an unhashable `role_preset`.
- **`cogs/music/cog.py`** — `!fix` no longer wipes the live queue/loop/volume:
  its own disconnect echo could arm a deferred `cleanup_guild_data`; it now only
  runs that cleanup when the bot is genuinely disconnected.
- **`native_dashboard/src/database.rs`** — the connection-reuse probe used
  `execute("SELECT 1")` (always `Err(ExecuteReturnedResults)`), so the cached
  connection was discarded and the DB reopened on every dashboard call → switched
  to `query_row`.
- **`utils/media/ytdl_source.py`** — search queries like `OST: theme` /
  `C418: Sweden` were wrongly rejected as URLs (`urlparse` scheme); now gated on
  `"://"`.
- **`utils/database/database.py`** — connection-setup PRAGMAs moved inside the
  try so a transient failure can't leak a counted connection / drift the pool
  gauge.
- Plus: `api_handler.py` perf-metric clock mismatch (`time.time()` →
  `perf_counter()`); `dashboard_chat_claude.py` retry-prefill trailing-whitespace
  400; `ws_dashboard.py` `_client_inflight` leak; `ai_cache.py` checkpoint lock;
  `long_term_memory.py` cache-only filter; `memory_consolidator.py` empty-summary
  data-loss guard; `rag.py` index-built flag; `spotify_handler.py` null album;
  `error_recovery.py` `headers=None` crash; `self_healer.py`
  `psutil.TimeoutExpired`; `media_rust.py` base64 `validate=True`;
  `native_dashboard/src/main.rs` error-log delimiter; and the
  `inspect_dashboard_memories` SQL column.

## Dashboard (Tauri 2 + TypeScript) fixes

- **Streaming survives a conversation switch** — leaving a chat mid-stream and
  returning now replays the in-progress answer (buffered per-stream instead of
  living only in the DOM); the response stream is preserved rather than abandoned.
- **Uptime ticks at the chosen interval** — the `status` cache TTL was a fixed
  1.5 s, so a 1 s auto-refresh reused stale status and uptime jumped 0→2→4→6. The
  TTL now tracks the refresh interval.
