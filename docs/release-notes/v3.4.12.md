# v3.4.12 — Full-repo line-by-line audit (round 5): 2 high-severity bug fixes + 144 fixes across all stacks & dashboard rebuild

**Date:** 2026-06-17

Continuing the v3.4.8–v3.4.11 line-by-line audit series. Every non-test source
file across all four stacks (Python, TypeScript, Rust, Go) plus the hand-authored
UI and build/config was re-read **line-by-line** — grouped into **49 file-groups**,
each read by a fan-out auditor and then **independently re-read by an adversarial,
refute-by-default verifier** (which both adjudicates the first pass and catches
what it missed). **98 agents** produced **200 confirmed findings** (**0 critical,
2 high, 18 medium, 140 low, 40 improvement**); 57 were rejected as false positives
and 15 are uncertain (need runtime/external confirmation). The 2 high-severity
bugs were fixed and verified by hand; a 96-agent fan-out then applied **142 more
fixes across 96 files** and **skipped 60** (re-judged false-positive, intentional
design, or too risky/subjective — each with a recorded reason). No features
changed; no security constraints weakened (several were strengthened). Every suite
stays green.

## High-severity fixes

- **Go URL-fetcher backend was silently disabled out of the box**
  (`utils/web/url_fetcher_client.py`). The client attaches an SSRF-safe resolver
  to its session, but aiohttp only skips the resolver for *literal-IP* hosts — so
  the default `localhost:8081` health check resolved to `127.0.0.1` and was
  blocked by the resolver's own loopback rule, so `_service_available` could never
  become `True` and every fetch silently fell back to aiohttp (the Go path was
  dead code on default config). Fixed with a service-host-aware resolver
  (`_ServiceHostResolver`) that exempts the configured trusted host *by name*
  while keeping DNS-rebind protection on the user-URL fallback path. Also fixed a
  co-located `__aenter__` `ClientSession` leak on startup failure.
- **Migration re-run duplicated NULL-`message_id` history**
  (`scripts/maintenance/migrate_to_db.py`). The `ON CONFLICT` target is partial
  (`WHERE message_id IS NOT NULL`); legacy rows rarely carry a `message_id`, so
  they sit outside the unique index and were re-`INSERT`ed on every re-run — and
  re-running after a partial failure is an explicitly supported recovery path,
  so a second invocation silently doubled a channel's history. Now idempotent:
  NULL-id candidates are deduped against rows already migrated for the channel
  (a multiset keyed on role/content/timestamp, decremented so legitimate in-file
  duplicates are preserved).

## Notable fixes by area (highlights from the 142)

- **Persistence / dashboard** — the 6 dashboard-message mutators
  (edit / delete / delete-after / pin / like / edit+truncate) now schedule the
  on-disk per-conversation JSON export, so `dashboard_chats/<id>.json` no longer
  diverges from the DB after an edit; dashboard exports share (and respect) the
  AI-history 20-task budget (`utils/database/database.py`).
- **Async / resource** — `close_pool` now awaits the cancelled WAL-checkpoint
  task before tearing down the pool (no orphaned semaphore slot / "Task was
  destroyed" warning); cache `cleanup_expired()` runs off the event loop via
  `asyncio.to_thread`; the L2 flush re-snapshots pending futures so shutdown
  durability also covers late submissions (`cogs/ai_core/cache/ai_cache.py`).
- **Reliability / observability** — a failed token-usage persist batch now logs
  the dropped token totals for billing reconciliation (`token_tracker.py`); RAG
  search degrades gracefully on any sqlite/OS error instead of only `TimeoutError`
  and reports fractional `age_days` (`cogs/ai_core/memory/rag.py`).
- **Go services** — `health_api` waits for the graceful-shutdown drain before the
  process exits (in-flight `/metrics/*` and `/health/service` writes are no longer
  truncated); `url_fetcher` recovers panics inside `FetchBatch` worker goroutines,
  which run outside chi's `Recoverer` — a crafted URL in a batch could otherwise
  crash the whole process (DoS hardening).
- **Rust (Tauri)** — production bot-path resolution canonicalizes **once** and
  binds the `bot.py` existence check, the allowed-root confinement check, and the
  spawned path to that single resolution, closing a same-user TOCTOU window on the
  user-writable `bot_path.txt` marker; dotenv parsing strips at most one matched
  quote pair (`native_dashboard/src/main.rs`).
- **Frontend security** — the SVG `data:` URI blocklist is no longer bypassable
  via the comma form `data:image/svg,…` (`native_dashboard/src-ts/shared.ts`).
- **Tooling / process** — `dev_watcher` no longer holds its lock across the
  multi-second crash-retry sleep (which stalled the observer); `bot_manager` uses
  the hardened exact-basename process match; `reindex_db` view-detection no longer
  over-matches unrelated views and guards index-name collisions; `install_all.ps1`
  no longer aborts on Windows PowerShell 5.1 (the `Tls13` enum member).
