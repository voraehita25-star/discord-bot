# v3.4.10 — Audit round 3, documentation-accuracy pass & dashboard rebuild

**Date:** 2026-06-16

Maintenance / hardening release continuing the v3.4.8–v3.4.9 line-by-line audit
series. Every non-test source file (**173 files / ~83k LOC** — Python, Rust, Go,
TypeScript) was read line-by-line and put through **two** independent adversarial,
refute-by-default verification passes; the **53 verified findings** (all
low/medium — **zero critical/high**) are fixed. A separate
**documentation-accuracy audit** cross-checked every living doc against the real
code and corrected **37 drifted statements** across 15 files (one flagged change —
the dashboard image cap — was a false positive and reverted, since the documented
10 MB is the binding backend limit, pinned by `test_doc_consistency`). The native dashboard
was rebuilt and a fresh NSIS installer ships with this release. No features
changed; no security constraints weakened. Every suite stays green.

## Code fixes (53 verified findings)

165 raw findings → 127 confirmed (adversarial pass 1) → 54 vetted (strict
gatekeeper pass 2) → 53 applied. One finding was reverted (a cosmetic tz-aware
`datetime` change that broke a tested naive-datetime contract); two high-effort
mediums — an ffmpeg `ProcessPoolExecutor` rewrite and a music-settings schema
change — were deferred as too invasive for a patch.

Highlights:

- **Dashboard user-context cache no longer serves stale data after an
  invalidation** — a generation counter guards the write so a concurrent document
  upload / profile edit during DB I/O can't be clobbered, and the TTL is measured
  from write time (`cogs/ai_core/api/dashboard_common.py`).
- **`init_schema` is serialized** via a lazily-created schema lock + double-check,
  so two concurrent `health_check` recoveries can't run migrations against the
  same DB file at once (`utils/database/database.py`).
- **Stale `pause_start` reset on every queue advance** — skipping a paused track
  no longer corrupts resume / now-playing elapsed time (`cogs/music/cog.py`).
- **Failed slash commands are now recorded in metrics**, and `!sync` no longer
  misreports a successful sync as an error (`bot.py`).
- **Empty `{"text": ""}` history blocks are dropped** before the Claude SDK path
  (the API rejects them); webhook usernames are clamped to Discord's 80-char
  limit; read-only AI-tool permission refusals are reported with `is_error=True`.
- **Resource-leak / robustness fixes**: per-conversation temp-dir cleanup,
  auth-failure bucket pruning, `_OVERLIMIT_LAST_WARN` bounding, Spotify
  auth-manager session close, Pillow decompression-bomb guard, `db is None`
  analytics fallbacks, document-extractor page-count guard.
- **Go** `/metrics/push` now rejects a name/type mismatch with `400`; **Rust** WS
  port parsed as a `u16` with an `8765` fallback; **TS** `clearLogs` race fix,
  attachment-clear-only-on-success, LaTeX-after-bold ordering, provider
  persistence, `downloadExport` input narrowing.
- **+ test fixtures** updated for the new schema lock; the full suite stays green.

## Documentation accuracy (37 corrections across 15 docs)

Every living doc was cross-checked against the code (historical snapshots left
untouched):

- **`GEMINI_API_KEY` is decoupled from `CLAUDE_BACKEND`** — semantic RAG
  embeddings work on **any** backend incl. the default `cli` (corrected in README,
  DEVELOPER_GUIDE, INSTALL — was wrongly "only when `CLAUDE_BACKEND=api`").
- **`HEALTH_API_TOKEN` is required (fail-closed `503`)** for the Go health-API
  write endpoints — now documented in `go_services/README.md`.
- **`DASHBOARD_ALLOW_UNRESTRICTED` is not a WS-auth bypass** — it gates the
  unrestricted creative-writing prompt mode, not WebSocket authentication
  (corrected in TROUBLESHOOTING).
- Dashboard bot-path resolution order; DB-schema
  indexes (`ai_analytics`, `entity_memories`); `conversation_summaries` is created
  at migration time; over-limit warning cooldown is **10 min**; `cut-release` now
  ships an installer; and the root `FINDINGS.md` snapshot is marked
  resolved / historical.
- Volatile numbers (version, test/file counts, ruff version, module line counts)
  re-synced by `scripts/maintenance/sync_doc_stats.py`.
