# v3.4.11 — Toolchain & dependency upgrade (Python 3.14.6), audit round 4, persona-loader fix & dashboard rebuild

**Date:** 2026-06-17

A maintenance / upgrade release continuing the v3.4.8–v3.4.10 line-by-line audit
series, paired with a full toolchain + dependency upgrade to the latest stable
versions. Every non-test source file (**176 files** — Python, Rust, Go,
TypeScript) was read line-by-line across 36 parallel agents and put through an
adversarial, refute-by-default verification pass; the **53 verified findings**
(all low/medium — **zero critical/high**) were triaged: **50 fixed**, 1 confirmed
a false positive, 1 deferred (display-only), 1 left as intentional design
(refuted). No features changed; no security constraints weakened. Every suite
stays green.

## Code fixes (audit round 4 — 53 verified findings, 50 applied)

51 findings from the main fan-out + 2 native-Tauri re-run findings → 53 verified
(adversarial, refute-by-default) → **50 applied**. Highlights:

- **API failover correctness** — `call_claude_api` recorded success BEFORE the
  empty-content check, so a `200`-but-empty endpoint never tripped the circuit
  breaker or failed over (silent blank replies); success bookkeeping is now
  deferred until non-empty content, with a soft failure on exhausted-empty
  (`cogs/ai_core/api/api_handler.py`).
- **Channel-permission targeting** — `cmd_set_channel_perm` had no duplicate-name
  guard and could set overwrites on the wrong same-named channel (e.g. exposing a
  private `general`); now ID-first with a duplicate-name abort, mirroring
  `cmd_delete_channel` (`cogs/ai_core/commands/server_commands.py`).
- **Persona loader robustness** — `cogs/ai_core/data/__init__.py` did an
  all-or-nothing import on the new canonical symbol names, so a persona file using
  the legacy names silently fell back to the generic example; it now resolves each
  symbol per-name (new → legacy → safe empty default), so a real persona loads
  correctly without injecting any operator-undefined slot.
- **Concurrency / resource** — backoff-state eviction can no longer orphan an
  in-flight retry's state; `retry_async`/`calculate_delay_sync` clamp the adaptive
  delay to `max_delay`; `_get_channel_lock` eviction can't drop a just-returned
  lock; a per-user lock unifies `forget_fact`; pre-warmed `claude -p` children are
  reaped; the WAL-checkpoint task is cancelled on pool close.
- **Robustness / correctness** — smart-trim budget warning, L2-cache rowcount,
  post-set-hook torn read, GIF canvas-size bomb guard, structured-logger
  redaction filter, UTC token buckets, document-extractor text-path size guard,
  MCP exception redaction, `format_duration` negative clamp, migrations
  comment-stripper ordering, `view_db` connection close, app-command Sentry
  capture, Windows-SIGTERM future logging.
- **Go** redirect-hop hostname denylist (metadata-SSRF defense-in-depth);
  **Rust (Tauri)** `stop()` child-handle reap + Windows 8.3 short-path
  canonicalize + entry-script-only test-process exclusion; **TS** scroll-badge
  counts messages, not stream chunks.
- **5 findings re-fixed "the right way"** after first-pass attempts broke tested
  contracts: health-client now buffers-when-unavailable (bounded by cap),
  memory-consolidator drains a channel's backlog across capped passes per cycle,
  dev-watcher runs the health-check sleep outside `_lock`, and spotify
  album-pagination is wrapped in a single retry-safe closure (mirrors
  `_handle_playlist`).
