# v3.4.3 — Two-pass audit fixes, dead-code removal & a big test-coverage push

**Date:** 2026-06-03

Patch release built on a second read-every-line audit of the entire non-test
source tree (**164 files / 73,524 lines** across Python / Rust / TypeScript / Go),
a two-pass adversarial bug-fix sweep, removal of confirmed dead code, and a large
test-coverage expansion. The Python suite grew from the v3.4.2 baseline of
**3,143** to **4,721 pass / 2 skip** (110 files); dashboard **190 vitest** +
**70 Playwright** + strict `tsc` clean; `cargo check` clean on both Rust crates.
No new features and no breaking changes.

## Correctness, security & reliability (35 verified fixes)

Every non-test file was read line-by-line by a fan-out of reviewers; each raw
finding was then adversarially re-verified against the real code (the first
verifier pass errored on a batch under load, so all unjudged findings were
re-run). Of 132 raw findings, **35 were confirmed real** (2 medium, 33 low) and
97 were rejected as false-positive / intentional design / unreachable.
Highlights:

- **Medium — `cogs/ai_core/api/dashboard_chat_claude_cli.py`**: the first-turn
  `_track_session` write happened *after* the `finally` released the
  per-conversation lock (with awaits in between), defeating the in-lock re-read
  defense — two concurrent first turns could spawn a second fresh CLI session and
  orphan the first. The session id is now recorded while the lock is still held.
- **Medium — `scripts/dev_watcher.py`**: the hot-reload watcher reset
  `consecutive_crashes` to 0 on every launch regardless of the health check, so a
  bot that crashed immediately retried forever (the `max_crash_retries` cap was
  never reachable). The reset is now gated on a passing health check, and the
  max-retries branch clears the dead process handle so it stops re-detecting the
  same crash every tick.
- **`cogs/ai_core/media_processor.py`** — on a GIF→MP4 encode timeout the buffer
  was closed while the worker thread was still writing into it, raising inside the
  abandoned thread and leaking the ffmpeg temp file every timeout. The buffer is
  now closed only on the success path.
- **`cogs/ai_core/storage.py`** — `get_all_channel_ids` / `get_all_channels_summary`
  / `get_channel_history_preview` caught only `OSError`, so an `aiosqlite.Error`
  (not an `OSError` subclass) escaped instead of degrading to `[]`. Now catch both.
- **`cogs/ai_core/api/dashboard_chat*.py`** — the `conversation_id` validator used
  `$`, which matches before a trailing newline; switched to `\Z` across all five
  handler sites. Also: the history-strip now drops the exact saved row (not "the
  last user row"), so a failed save no longer silently discards the previous turn,
  and a corrupt attached image now sends a ws error frame instead of vanishing.
- **`cogs/ai_core/cache/ai_cache.py`** — `set()` evicted before reading the
  existing entry, so refreshing a hot key at capacity could drop its own
  carried-forward hits/TTL; eviction now runs only for genuinely new keys.
- **`utils/database/database.py`** — `health_check` probed `SELECT 1` (passes on a
  tableless recreated DB); it now probes a core table and rebuilds the schema on
  the recovery path. The debounced JSON export discards its pending key only after
  the write completes, so an in-window save can't spawn an overlapping writer.
- **`utils/reliability/`** — `error_recovery.extract_retry_after` now honors an
  HTTP-date `Retry-After` (not just delta-seconds); `shutdown_manager` counts
  handlers cancelled on budget-exhaustion so the summary totals add up.
- **`utils/media/ytdl_source.py`** — an uppercase-scheme direct URL (`HTTP://…`)
  was mis-wrapped as a `ytsearch:` term; the scheme check is now case-insensitive.
- Plus low-severity fixes in `cache/token_tracker.py` (tz-aware min/max),
  `core/performance.py` (dedup-key status-block strip), `memory/entity_memory.py`
  (`age == 0` shown), `memory/history_manager.py` (`smart_trim` cap),
  `memory/rag.py` (batch-embedding dimension guard), `music/cog.py` &
  `music/queue.py` (per-guild save isolation + DB-load queue cap),
  `native_dashboard/` (`app.ts` avatar-crop guard + failover re-request on
  reconnect, `context-window.ts` redundant write, `document-attach.ts` empty-file
  feedback, `bot_manager.rs` PID-name verification), and several maintenance
  scripts (`rollback_migration.py`, `migrate_to_db.py`, `find_unused.py`,
  `inspect_db.py`, `validate_ipc.py`, `validate_runtime.py`).

## Dead code removed

Four confirmed-unused items deleted (tests only, no behavioral change to live
paths): the never-used `Database._export_lock`, `EntityMemoryManager.delete_entity`
(no callers), both `quick_check` methods in `processing/guardrails.py` (no
production callers; superseded by `validate()`) plus their tests, and the
unreachable `_failover_cleanup` branch in `chat-manager.ts` (no backend ever emits
that frame).

## Test-coverage expansion (~1,580 new tests)

- **Regression tests** for all 35 fixes above, so each bug is locked in (several
  fail on the pre-fix code, e.g. the dedup-key collision and the health-gated
  crash counter).
- **New dedicated suites** for previously-untested modules: `migrations.py`,
  `url_fetcher_client.py`, `url_safety.py`, `core/performance.py`, `dev_watcher.py`,
  `imports.py`, `dashboard_config.py`, `bot_manager.py`, `ffmpeg_path.py`,
  `character_tags.py`, `ffmpeg`/utils helpers.
- **Deepened coverage** of the security/memory/media subsystems
  (`url_fetcher.py` 33→94%, the RAG + long-term/entity/consolidator memory stack,
  `spotify_handler.py`, `ytdl_source.py`, `api_failover.py`).
- **Discord cogs** driven to ~100%: `ai_cog.py`, all command cogs, and
  `music/views.py` / `queue.py` / `url_safety.py` / `utils.py` at 100%;
  `music/cog.py` at 99.5% (the residual lines are defensive/near-unreachable
  guards). All tests-only — no source changed and no `# pragma: no cover` used.
- `coverage` + `pytest-cov` added to the dev toolchain (`make test-cov`);
  `cogs/` + `utils/` line coverage rose from ~55% to ~63% (production modules ~62%).
