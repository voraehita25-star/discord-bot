# v3.4.14 — Faust persona wiring: roleplay format (Discord + dashboard), unrestricted text fix, and CLI identity override

**Date:** 2026-06-18

A full trace of the Faust/roleplay/unrestricted persona system (two parallel
systems — `faust_data.py` for Faust, `roleplay_data.py` for the RP-server world —
bridged by a legacy/canonical name-aliasing loader) surfaced three operator-
authored persona pieces that were defined but **never reached a model**, plus a
CLI-backend identity leak. This release wires them up and fixes the leak. The
operator persona file `cogs/ai_core/data/faust_data.py` is **not modified** (it
carries a DO-NOT-EDIT marker); everything is wired from the consumer side.

## Roleplay format (`FAUST_ROLEPLAY`) — was dead, now active

`FAUST_ROLEPLAY` (the `>`-prefixed action-narration rules, "don't control other
users", `{{Character}}` multi-char syntax) was a re-exported constant imported by
**no** consumer — editing it did nothing. It is now appended to the base persona:

- **Discord guild channels** (`session_mixin.py`) — non-RP guilds get the
  format; **DMs keep the plain persona** (no RP actions, matching
  `FAUST_DM_MODE`'s intent); the dedicated RP server keeps its own
  `ROLEPLAY_PROMPT`. A cache-correction path lets already-open channels pick it
  up without a restart.
- **Dashboard `faust` preset** (`dashboard_config.py`) — applies to all three
  dashboard backends (Gemini / Claude SDK / Claude CLI) since they all read
  `preset["system_instruction"]`.

A `!= FAUST_INSTRUCTION` guard prevents duplicating the persona on setups where
the loader defaulted `FAUST_ROLEPLAY` back to the base persona.

## Unrestricted text (`FAUST_SANDBOX`) — Discord `!unrestricted` was a no-op

The operator file defines `FAUST_SANDBOX` but not `UNRESTRICTED_MODE_INSTRUCTION`
(the canonical name the loader looks up), so the latter resolved to `""` — the
owner-only Discord `!unrestricted` toggle's gate fired but injected **nothing**.
`session_mixin.py` now uses `UNRESTRICTED_MODE_INSTRUCTION or FAUST_SANDBOX`, so
the toggle injects the operator's authored text. Scope is Discord-only; the
dashboard keeps its own `GENERAL_UNRESTRICTED_FRAMING` fallback unchanged.

## CLI backend identity override — Faust no longer greets as a coding assistant

The `cli` backend (the default) delivers the persona via
`--append-system-prompt-file`, which Claude Code **prepends** its built-in
software-engineering-assistant system prompt to — so the model leaked that
identity and would greet as a coding/repo assistant instead of the persona. (The
SDK and Gemini backends *replace* the system prompt, so they were already
correct.) Both CLI prompt builders now lead with a persona-agnostic
`_IDENTITY_OVERRIDE` block (system-level for the dashboard CLI via
`_build_system_prompt`, in the `# System` body section for the Discord CLI) that
tells the model to drop the "Claude Code"/coding-tool identity and adopt **only**
the persona below, and not to volunteer code/repo/dev help unprompted. It is
persona-agnostic so it is correct for both the character and general presets.

> If a full reset is ever preferred over the override, Claude Code 2.1.179
> supports `--system-prompt-file` (replace), but that drops the tool scaffolding
> (e.g. Read-for-images) the append path intentionally preserves.
