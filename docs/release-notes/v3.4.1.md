# v3.4.1 ‚Äî Security + correctness hardening, dashboard deep-audit & a11y

**Date:** 2026-05-31

Patch release built on a read-every-line audit of the whole repo and a deep
re-audit of the native dashboard. Verified fixes, accessibility work, and
documentation accuracy. Also carries the working-tree refinements accumulated
since v3.4.0 (Claude CLI/chat paths, dashboard WebSocket backend, and their
tests). Python suite **3,379 pass / 1 skip**; dashboard **189 vitest** + strict
`tsc` clean.

## Security (1)

- **Log secret-redaction gap fixed** (`utils/monitoring/logger.py`). The
  keyword branch (`key=` / `token=` / `password=` / `authorization:` / `bearer`)
  chained two lookbehinds at the same offset that contradicted each other, so it
  matched **nothing** ‚Äî opaque, non-prefixed secrets (e.g. the health-API token)
  leaked into log files. Rebuilt the assertion into one fixed-width lookbehind,
  added `bearer`, and covered the JSON `"key":"‚Ä¶"` form. Proven with a direct
  `_redact_sensitive()` test (5 secret forms redacted, `monkey: <hex>` left
  intact).

## Correctness & reliability (8)

- **`utils/fast_json.py`** ‚Äî corrected the false "orjson raises on NaN" comment
  (orjson emits `null`); the stdlib `json_dumps_bytes` fallback now uses
  `allow_nan=False` for parity with `json_dumps`.
- **`utils/web/url_fetcher.py`** ‚Äî dropped `enable_cleanup_closed=True` (a no-op
  that warns on Python 3.12.7+/3.14; the underlying asyncio leak is fixed upstream).
- **`cogs/ai_core/logic.py`** ‚Äî RAG retrieval now degrades on any backend error
  (was `except OSError` only, so a FAISS/numpy error aborted the whole turn).
- **`cogs/ai_core/memory/entity_memory.py`** ‚Äî `EntityFacts.to_prompt_text` caps
  each field + bounds custom rows, so an over-long/injected fact can't blow the
  context budget.
- **`cogs/ai_core/ai_cog.py`** ‚Äî `_resolve_prefix_tuple` filters empty prefixes
  (an empty string made `startswith` match every DM/mention and silently drop it).
- **`cogs/ai_core/commands/server_commands.py`** ‚Äî AI tool error echoes that
  interpolate raw user/role/channel names now send `AllowedMentions.none()`
  (mention-injection hardening, consistent with the codebase's mention posture).
- **`scripts/maintenance/count_cli_sessions.py`** ‚Äî de-duplicated the survey so
  the now-identical bot/Claude path encoders no longer double-count session files.
- **`scripts/install_all.ps1`** ‚Äî Python fallback installer pinned a 404-ing
  alpha (`3.14.0a3`); bumped to stable `3.14.4`.

## Native dashboard (Tauri 2 + TypeScript)

A deep re-audit across BUG / MISSING / ENHANCE lenses, then fixes + a rebuilt
release `.exe`.

- **Streaming correctness** ‚Äî `regenerateAfterEdit` and the AI-edit submit now
  set the streaming gate (closes a double-send window); the `error` frame clears
  `streamingConversationId`; `chunk` / `thinking` / `finalize` casts are
  `typeof`-guarded so a missing field can't inject `"undefined"` or throw and
  strand the input.
- **WebSocket self-heal** ‚Äî the client no longer dies permanently after 5
  reconnect attempts; it keeps retrying at the capped 30 s interval (warning the
  user once) and recovers automatically when the bot returns.
- **New `info` / `status` handlers** ‚Äî backend `info` frames were silently
  dropped; both now surface as toasts.
- **Documents chip** ‚Äî attached files show a `üìé filename` chip under the message
  (session-scoped; the backend stores docs at conversation scope).
- **Accessibility** ‚Äî `<html lang="ko">`, labelled nav + `aria-current`, a focus
  skip-link, `prefers-reduced-motion` for the sakura animation, and avatar
  `FileReader` / image `onerror` handlers.
- **Robustness & tooling** ‚Äî `showToast` no longer suppresses error/warning
  toasts when notifications are off; context-window usage is validated on write
  (no more persisted `NaN%`); added a `typecheck` script; removed 8 unused
  symbols and enabled `noUnusedLocals` / `noUnusedParameters`.
