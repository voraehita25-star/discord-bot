# v3.4.5 — Full repo line-by-line audit: bug fixes + hardening

**Date:** 2026-06-08

Quality release. Every non-test source file (171 files / ~75k lines across Python,
Rust, TypeScript, Go) was reviewed line-by-line with adversarial verification, and
each finding was re-checked against the real code before any change. The result is a
broad sweep of correctness fixes and safe hardening/cleanup — **no breaking changes,
no behavioral changes to the happy path**, and every test suite stays green. False
positives and behavior-risky "fixes" were deliberately *not* applied (see below).

## Notable bug fixes

- **Dashboard profile preferences were silently dropped** (`api/dashboard_handlers.py`).
  The UI sends `preferences` as a free-text string, but the handler only processed a
  `dict`, so the value was stored as `NULL` every save (and a dict, had any client
  sent one, would have raised on the SQLite TEXT bind). Now a string is sanitized and
  persisted; a dict is sanitized and JSON-serialized — always `str | None` per the
  column contract.
- **Music volume loaded from the queue sidecar was never clamped** (`music/cog.py`).
  A corrupt/hand-edited `data/queue_<gid>.json` with `"volume": 99` or `NaN` flowed
  straight into the `PCMVolumeTransformer`. Now clamped to `[0.0, 2.0]` with a finite
  check, matching the `!volume` command.
- **CLI `--resume` session not reset on `link`/`move` memory** (`ai_core/ai_cog.py`).
  In the default CLI backend, the target (and, for move, source) channel kept
  replaying stale server-side context that didn't reflect the copied/deleted history.
  Now mirrors `reset_ai`'s session reset.
- **Audit-log JSONL hash-chain race** (`utils/monitoring/audit_log.py`). The running
  head was read, then committed *after* an `await`, so two concurrent fallback writes
  could fork the tamper-evidence chain. The read→write→commit is now one critical
  section under the fallback lock.
- **Decompression-bomb guard bypass in Fill resize** (`rust_extensions/media_processor`).
  An extreme-aspect input (e.g. 16384×1) sailed past the 100 MP input check but
  exploded the Fill intermediate; the intermediate is now bounded by the same cap.
- **Chat code blocks with `$` were mangled** (`native_dashboard/.../formatter.ts`).
  `$HOME`, jQuery `$(...)`, PHP `$var`, etc. inside fenced code were consumed by the
  LaTeX pass. Code is now extracted *before* the LaTeX passes.
- **Prism highlighter loaded grammars without prerequisites** (`.../chat/prism.ts`).
  The first js/ts/go/c/cpp/etc. block of a session silently failed to highlight; a
  dependency map now loads base grammars first, with verify-on-load.
- **Health-API auth crashed instead of returning 401** (`utils/monitoring/health_api.py`).
  A non-ASCII `Authorization` header made `hmac.compare_digest` raise `TypeError`
  (fail-open-to-crash). The comparison is now done on bytes.
- **JSON-fallback history cache served stale reads** (`ai_core/storage.py`). The
  JSON save path never invalidated the in-memory cache (the DB paths did).

Plus many lower-severity correctness fixes: `if local_id:`/`if limit:`/`threshold or`
falsy-zero bugs, dedup-key leak + quota silent-bypass on the chat error path,
`smart_trim` dropping a middle band of history, non-deterministic member matching,
`@EVERYONE` case-folding, `ensure_system_prompt_file` returning a non-existent path,
the `_handle_album` Spotify teardown race, and more.

## Hardening & cleanup (selected, ~75 changes)

- Lenient env parsing for the Claude CLI backend (a malformed numeric override no
  longer aborts import); blank security-gate env vars fall back to their safe default.
- Pre-warmed `claude -p` process sweep so an abandoned conversation's ~150 MB process
  is reclaimed instead of leaking for the bot's lifetime.
- `NODE_OPTIONS` allowlist now exact-matches the valueless flag; `_deny` typed
  `NoReturn`; rate-limiter `set_channel_limit` honors the bucket cap; LRU `move_to_end`
  on cache refresh; deterministic timestamps; Win64 `GetStdHandle` restype fix.
- Doc/comment corrections (stale comments, dead-code notes), `__all__` for
  `utils/monitoring`, and consistency fixes across the monitoring/reliability modules.

## Deliberately NOT changed (verified real but skipped)

To avoid regressions on a production bot, several findings were intentionally left
alone: the `after_seek` race (the obvious one-line guard would break normal seek; a
correct fix needs an audio-thread refactor), the streaming "dead-code" salvage branch
(both suggested fixes regress long responses), the migrations SQL comment scanner, and
a handful of low-value/behavior-risky items. A `document_extractor` "decode" change was
reverted because it contradicted a tested contract.
