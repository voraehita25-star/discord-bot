# v3.4.9 — Audit round 2, documentation-accuracy pass & self-updating doc stats

**Date:** 2026-06-15

Maintenance / hardening release, following v3.4.8's first line-by-line pass. Every
non-test source file (**174 files** — Python, Rust, Go, TypeScript) was read
line-by-line again and audited with adversarial, refute-by-default verification;
the **21 confirmed** (low/nit) findings are fixed. A separate
**documentation-accuracy audit** cross-checked every living doc against the real
code and corrected **25 drifted statements**. Finally, a new tool makes the
volatile doc numbers **update themselves**, so they stop drifting between
releases. No features changed; no security constraints weakened. Every suite
stays green.

## Code fixes (21 verified findings)

- **Claude CLI transcript cleanup no longer leaks on dotted paths.**
  `_encode_claude_project_dirname` now matches Claude Code's real
  `[^A-Za-z0-9] → -` encoding (it hand-listed `: \ / space _` and missed `.`),
  so session-file delete / LRU eviction target the folder the CLI actually wrote
  — empirically verified against the installed CLI.
- **WebFetch isn't advertised on a turn where the argv withholds it.** On
  attachment (Read) turns the cacheable system prompt stops offering WebFetch
  (new `webfetch_enabled` gate), matching `_build_claude_argv`'s exfil-safety
  allow-list — no more confident calls to a denied tool.
- **Dashboard concurrency caps aligned.** The read-loop dispatcher gate (was 4)
  now matches the inner per-client cap (2), so a frame can't be spawned then
  rejected and silently burn a rate-limit slot.
- **Narrator intros split Thai-safely.** A new shared `_split_for_discord` helper
  (used by both the narrator path and the normal send) keeps Thai combining marks
  attached to their base char — the raw 2000-char slice could orphan one as a
  stray ◌ glyph.
- **Music seek clears the stale `pause_start`,** so a later pause/resume computes
  elapsed position from the right instant.
- **Token validator accepts valid legacy tokens.** `validate_token` floor
  70 → 59 (Discord doesn't guarantee a 70-char minimum) and now requires all
  three segments non-empty.
- **IPv6 url-fetcher host is bracketed** (`http://[::1]:8081`) so the
  allow-listed `::1` config reaches the Go service instead of silently failing
  over to aiohttp.
- Plus accuracy / logging fixes in `chat-manager.ts` (export Blob MIME),
  `ws-client.ts`, `history_manager`, `long_term_memory`, `logger`, `feedback`,
  `api_handler`, `roleplay_data_example`, `probe_ai_deep`, `convert_log015`.
- **+2 regression tests** (dot-encoding, `webfetch_enabled`); 1 updated to the
  corrected WebFetch contract.

## New: self-updating doc stats

`scripts/maintenance/sync_doc_stats.py` + `make docs-sync` / `make docs-check`
recompute every volatile number in the docs from the live repo — Python / vitest
/ Playwright test counts, test-file and source-file counts, the app version (from
`tauri.conf.json`), big-module line counts, the ruff version — and rewrite them
across all docs in one pass. It errors loudly if a registry pattern stops
matching (so a stale number can't hide), is best-effort when a toolchain is
absent, and `--check` guards drift in CI. This release's doc version bump (and a
`256 → 257` source-file count) were applied by the tool itself.
