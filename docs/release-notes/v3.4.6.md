# v3.4.6 — Dashboard AI History editor + repo-wide line-by-line audit

**Date:** 2026-06-13

Combined feature + quality release. It ships the Dashboard **AI History editor** and a
**Claude CLI backend overhaul** that had been in progress since v3.4.5, then layers a
fresh repo-wide line-by-line audit on top: all 168 non-test source files (~80k lines
across Python, Rust, TypeScript, Go) were re-reviewed with adversarial verification, and
every finding was re-checked against the real code before any change. **No breaking
changes, no happy-path behavior changes**, and every suite stays green.

## Dashboard & Claude CLI

- **Dashboard AI History editor** — browse / edit / delete / undo conversation history
  with live-session sync, plus a per-conversation history manager (`native_dashboard`
  `history-manager.ts`, `chat_manager_registry.py`, message-sync path).
- **Claude CLI backend overhaul** — delta-on-resume (only the new message goes out on a
  resumed server-side session), session self-heal on stale/errored sessions, transcript
  cleanup, and a `CLI_PROMPT_MAX_CHARS` over-limit choice flow.
- **Guardrails module removed** — `cogs/ai_core/processing/guardrails.py` is gone;
  unrestricted-mode gating now lives in the decoupled `processing/unrestricted.py`.

## Audit — verified bug fixes (selected)

Of 138 raw findings, **61 were confirmed** against the real code (77 rejected as
overstated); **55 were fixed**. Highlights:

- **Duplicate-instance race on bot start** (`scripts/bot_manager.py`). `auto_stop_existing_bot()`
  killed only the bot process, not the `start.ps1` restart-loop launcher, and never wrote
  `STOP_FLAG` — so the surviving loop re-spawned the old bot to race the fresh launch. Now
  mirrors `stop_bot()` (kill launcher + flag, clean up before relaunch).
- **Hot-reload silently disabled under a dot-prefixed ancestor dir** (`scripts/dev_watcher.py`).
  `_should_ignore` inspected the full absolute path, so a checkout under `~/.local`, `~/.cache`,
  etc. made the hidden-dir check match every event. Now scoped to the repo-relative path.
- **`verify_system.py` skipped first-party source** — the bare-basename `"data"` exclusion
  also pruned `cogs/ai_core/data/` (5 modules), so a syntax error there reported "passed".
  Anchored to the project-root dump only.
- **Alert cooldown lost-update race** (`utils/monitoring/alerting.py`). `_rollback_cooldown`
  promised a compare-and-set in its docstring but overwrote unconditionally, re-opening the
  window for a duplicate alert. Now returns the reserved timestamp and CAS-guards the rollback.
- **Self-healer singleton lock & log path were CWD-relative** (`utils/reliability/self_healer.py`)
  — two bots launched from different directories opened different lock files (no mutual
  exclusion). Anchored to the repo root like `PID_FILE`.
- **CLI token-usage `int(None)` crash** (`api/dashboard_chat_claude_cli.py`). A JSON-null
  usage field would raise `TypeError` after the reply was already streamed, hanging the UI;
  now `... or 0` like the SDK backend.
- **`anthropic.APITimeoutError` mis-recorded** (`api/api_handler.py`) — it subclasses
  `APIConnectionError`, not builtin `TimeoutError`, so SDK timeouts skipped the timeout
  metric path. Added to the timeout branch.
- **Model cost estimator** (`cache/token_tracker.py`) — added Opus 4.6/4.5, Fable 5, Mythos 5,
  and corrected the Opus-family fallback from the retired `$15/$75` (Opus 3) to `$5/$25`.
- **Mention sanitizer dropped the legacy `!`** (`ai_core/sanitization.py`); **circuit-breaker
  async path could wedge HALF_OPEN** (mirrored to the fixed sync path); **WS rate-slot leaked
  on concurrency-cap rejection** (`api/ws_dashboard.py`); **PIL resize flattened LA/PA/palette
  transparency to JPEG** (now routed to PNG); **DB write-lock held across a filesystem unlink**
  (moved outside the lock); and ~40 more lower-severity correctness/robustness fixes.

## Documentation accuracy sync

A separate doc-accuracy pass cross-checked every living doc against the current code and
fixed **16 verified inaccuracies** across 9 files: stale test counts, a deleted-module
reference (`guardrails`), wrong `make build-rust`/`build-go` comments, a `CLAUDE_BACKEND`
description implying `api` was required for Discord replies, a stale WebSocket frame-cap
figure (~43 MB → ~351 MB), a `check_db.py`→`inspect_db.py` schema-dump fix, an owner-command
rejection message that didn't exist in code, the RagEngine example (dimension 384→768, real
`add()` signature), and the `v3.4.1`→`v3.4.5` version drift in `INSTALL.md`. Historical
records (`docs/reviews/**`, release notes) were deliberately left untouched.

## Deliberately NOT changed (verified real but skipped)

To avoid regressions on a production bot, a few findings were intentionally left alone: the
`bot_manager` `cpu_percent` two-pass rewrite (would slow the common single-instance path),
the `performance_tracker.record()` lock (a real fix needs coordinated locking across ~6
reader methods on a hot path for non-load-bearing observability data), and the migrations
SQL string-literal scanner (latent only — no current migration triggers it; documented the
invariant instead of rewriting the critical startup splitter).
