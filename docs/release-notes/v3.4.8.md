# v3.4.8 — Repo-wide line-by-line audit: fixes, regression guards & dashboard rebuild

**Date:** 2026-06-14

Maintenance / hardening release. Every non-test source file in the polyglot tree
(**242 files** — Python, Rust, Go, TypeScript, PowerShell, SQL, docs, config) was
read **line-by-line** — not just linted — and audited with adversarial
verification. Of 155 confirmed findings the real user-facing bugs are fixed, the
long tail of low/nit issues cleared, and the impactful fixes are now pinned by
regression tests + a doc-consistency CI guard so they can't silently regress. No
features changed; no security constraints weakened. Every suite stays green and
the native dashboard was rebuilt to embed the frontend/backend fixes.

## Real bugs fixed

- **Attachments: a 30–60 MP image no longer aborts the whole batch.**
  `Image.DecompressionBombWarning` is a *sibling* of `DecompressionBombError`
  (neither subclasses the other), so the module's `filterwarnings("error", …)`
  raised a class that escaped the `except` tuples in `is_animated_gif`,
  `convert_gif_to_video`, and `process_attachments` — now caught in all three.
- **Music: a button-initiated pause kept the wrong elapsed time.** The
  `MusicControlView` pause button never set `pause_start`, so resume /
  `nowplaying` math drifted past the real audio position. Pause/resume
  bookkeeping is now shared (`mark_pause` / `mark_resume`) between the text
  command and the button.
- **Dashboard: a re-render no longer yanks a scrolled-up reader to the bottom.**
  `renderMessages()` captures scroll geometry before the DOM swap and only
  auto-scrolls when the reader was already near the tail (or streaming).
- **Health metrics: no data loss on a mid-flush failure.** `_flush_buffer_locked`
  re-queues only the *undelivered* tail of a chunked POST instead of the
  already-delivered front (which both lost the tail and double-counted the head).
- **Dashboard bot detection in dev mode.** `bot_manager.rs` now also matches a
  process by working directory (`with_cwd`), recovering the dev-watcher's
  relative-`bot.py` spawn that puts no absolute path in argv.
- **Startup: the clean-exit restart loop is bounded.** An immediate (< 10 s)
  clean exit now counts toward the restart limit so `start.ps1` can't hot-loop.
- **Docker image ran zero migrations.** The root `.dockerignore` excluded
  `scripts/`, dropping the migration subtree from the build context; the DB
  migration directory is re-included.

## Regression guards (so these stay fixed)

- New `tests/test_doc_consistency.py` pins documented values to their single code
  source of truth — `STREAMING_TIMEOUT_INITIAL`, `MAX_HISTORY_ITEMS`, the
  `ai_user` / `ai_guild` rate limits, the dashboard image cap, and the
  `.dockerignore` migration re-include. **Doc drift now fails CI immediately**
  instead of being rediscovered by the next audit.
- Behavioral regression tests for the decompression-bomb warning, the music
  pause/resume bookkeeping, the health-metrics partial-flush, and the dashboard
  scroll-preservation — each verified to fail if its fix is reverted.
- **15 new tests** total, all enforced by the existing CI jobs
  (pytest / vitest / Go / Rust / Playwright).

## Stale docs corrected (and now guarded)

`DEVELOPER_GUIDE.md` (`STREAMING_TIMEOUT_INITIAL` 30 → 120 s, `MAX_HISTORY_ITEMS`
2000 → 8000), `TROUBLESHOOTING.md` (`ai_user` / `ai_guild` 10/30 → 60/240
req·min⁻¹), `native_dashboard/README.md` (image cap 20 → 10 MB), and the
`repo-audit` skill's stale test baselines were all corrected to match the code.
