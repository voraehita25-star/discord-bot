## Notes / Cross-cutting

- **Token estimation is wrong for Thai content.** This is a Thai RP bot. Every cost estimate, every analytics summary, every `tokens_estimate` field returned by `get_realtime_stats` is wrong by ~30-40%. This affects user-visible dashboards and the `/ai_stats` command. Fix by using `claude_count_tokens` from the SDK or a per-language calibration table.

- **Three places define `setup` for sub-cogs but only one path is exercised.** `cogs/ai_core/ai_cog.py:1638` adds `AIDebug` and `MemoryCommands` directly. The `setup` functions in `commands/debug_commands.py:381` and `commands/memory_commands.py:262` are dead. **Risk:** if someone adds `cogs.ai_core.commands.debug_commands` to the load list, the cog gets registered twice. Either remove the dead setups or switch to extension-loading exclusively.

- **`flush_l2_pending` is unwired.** All the L2 future-tracking machinery in `ai_cache.py` (~30 lines of code) only matters during shutdown, but no shutdown handler ever calls it. On normal shutdown, in-flight L2 SQLite writes are dropped. Wire it from `bot.close` / `cog_unload`.

- **Mixed datetime tz-awareness in analytics.** `record_intent_feedback` uses naive `datetime.now()` while `log_interaction` uses `datetime.now(timezone.utc)`. Future code that compares them will TypeError.

- **Prompt injection surface.** `context_builder.AIContext.build_system_context` concatenates RAG/entity/state strings INTO the system prompt without sanitisation. URL content IS wrapped, others aren't. RAG retrieves prior user messages — a user can store a prompt-injection payload via `!remember` and have it appear in another user's RAG context.

- **Pricing in `token_tracker` is stale and provider-misclassifies non-Anthropic models as Gemini Flash.** Operationally important if costs are tracked.

- **Many sub-modules use `commands.command` (prefix) only — no slash commands.** Fine for owner-only debug commands, but `MemoryCommands.remember/forget/memories` are user-facing and would benefit from slash + autocomplete. Discord deprecates message-content intent gradually; future-proof by porting at least the user-facing commands.

- **Lock metadata in `MessageQueue` is fragile.** `_lock_times` is updated on acquire but never on release; ownership isn't tracked; force-release is intentionally disabled. If a coroutine crashes mid-processing, the channel is permanently locked. The previous review (`CODE_REVIEW_REPORT.md`) flagged this; still open.

- **No tests verify `find_similar` / `find_semantic_match` snapshot semantics.** Both walk the entire cache under the lock, then release and score. If a `set()` happens during scoring, the score is on stale data — acceptable, but no test pins the behaviour.

- **`InteractionLog` dataclass is defined but unused.** Symptom of half-finished refactor — either pass it to `log_interaction` or drop it.
