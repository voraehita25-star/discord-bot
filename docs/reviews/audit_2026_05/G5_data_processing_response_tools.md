## Notes / Cross-cutting

1. **Persistent dead code.** `processing/self_reflection.py` (443 LoC), `tools/tools.py`'s legacy `_*_webhook_cache` aliases, `prompt_manager.get_quick_response` / `get_system_prompt`, and `response_sender.send_typing` are all unused outside their own files or tests. Either wire them or delete. Carrying dead code obscures the real call graph.

2. **Storage durability is mixed.** `_save_history_json` for history is atomic+fsync (line 489-499 missing fsync though), but metadata write (`_write_meta`, line 507) is direct. The unrestricted-channels save (`guardrails.py:63-95`) is fully atomic+fsync — adopt that exact pattern across all storage writers.

3. **Cache mutation footgun.** `storage.load_metadata` returns shallow-copied or raw cache refs depending on hit/miss. `storage.load_history` correctly uses `copy.deepcopy` on both paths. Standardise.

4. **Tool format Gemini vs Claude.** `tool_definitions.py` exclusively builds Gemini-shaped schemas; the production model is Anthropic Claude. The Claude tool definitions live elsewhere (verify `dashboard_chat_claude*.py` or `api_handler.py`). If both paths exist, ensure they stay in sync — currently no shared source-of-truth means drift is inevitable.

5. **Substring matching for character names** (`media_processor.load_character_image`) and **substring matching for channel-history keywords** (`response_mixin.HISTORY_KEYWORDS`, `LIST_KEYWORDS`) both use naive `in` checks. Both will false-positive (e.g. "history" in "historical"). Migrate to word-boundary regex or token-set intersection.

6. **Pillow MAX_IMAGE_PIXELS not set** in `media_processor.py`. Other modules (`utils/media/media_rust.py`) set it explicitly. Add `Image.MAX_IMAGE_PIXELS = 30_000_000` and `warnings.simplefilter('error', Image.DecompressionBombWarning)` at module top to convert warnings into hard errors that the existing `except` clauses catch.

7. **Guardrails coverage gaps.** (a) `quick_check` only inspects 3 patterns, missing all Thai injection vectors and `bypass_safety`. (b) Repetition detection is gated by length thresholds that miss small-but-clearly-broken outputs. (c) Output sensitive-pattern redaction does not also apply to user input before storage.

8. **Permission model on `remember` tool** allows any guild member to write to long-term RAG memory. This is the highest-impact bug — RAG content is replayed into prompts, so a single user can poison the bot's memory persistently. Already filters obvious `[SYSTEM]` markers but not factual misinformation injection.

9. **Comment-vs-code drift:**
   - `roleplay_data_example.py:116` documents `display_name` values; `roleplay_data.py:286` documents `image_path` values; `media_processor.py:321` treats them as image paths. The example file is wrong.
   - `data/constants.py:49-51` claims limits are token-based; they're used as message counts in DB pruning.

10. **Untracked YAML schema.** `prompts/base.yaml` and `prompt_manager` have no formal contract. Add a load-time validation that the manager's required dotted keys exist.

11. **Async/blocking concerns.** No blocking I/O found in the async paths (all file IO routed through `loop.run_in_executor`). PIL operations (`Image.open`, `image.copy()`) are CPU-bound and run inline in `prepare_user_avatar` / `process_attachments` / `load_character_image` — for small images this is acceptable, but a 30MP image decode blocks the event loop. Consider `to_thread` for image decoding.

12. **Discord 2000-char split** is correctly handled in `response_sender.split_content` with code-fence carry-over (good), but the truncation notice when `max_chunks=20` is hit isn't propagated up the SendResult API.
