## Notes / Cross-cutting Observations

- **Multiple silent fallback layers for guardrails**: `imports.py` defines fail-open `validate_response` if guardrails unavailable; `__init__.py` defines `validate_response = None` for the same condition. Two systems with different semantics ŌĆö callers can't reliably tell which one they're getting. Pick ONE strategy.
- **Many magic numbers**: 5 emoji limit, 600s cleanup interval, 8000 lore length cap, 30s save timeout, 60s sync cooldown, 9090 metrics port. Centralize in a `constants.py`.
- **Hardcoded Thai messages everywhere**: bot.py error messages, voice.py user-facing text, fallback_responses.py, etc. The `utils.localization` module exists but is bypassed in many places.
- **Dual sources of truth for paths**: `config.py` defines `data_dir`/`temp_dir`/`logs_dir` anchored to project root; `bot.py:bootstrap()` uses relative `./temp`, `./data`, `./logs`. They can diverge if CWD differs from project root.
- **Class-attribute-as-instance-state pattern**: `start_time`, `_health_task`, `_metrics_started`, `_shutdown_task`, `_default_executor` declared as class attributes with default values, then mutated as instance state. Subtle and bug-prone ŌĆö use `__init__`.
- **No locks on global mutable state**: `feature_flags` dict, `fallback_system._fallback_count`/`_recent`, etc. are mutated from async code without locks. Mostly safe due to single-threaded asyncio loop, but if any threadpool task touches them, races are possible.
- **`global bot` rebinding** in `run_bot_with_confirmation` is fragile: many places reference `bot` directly via global lookup (e.g. health hooks). After rebind, old hooks may still reference the old bot.
- **`graceful_shutdown` is called from multiple paths** (signal handler, Ctrl+C confirmation, signal threadsafe submit). State machine is implicit ŌĆö consider explicit shutdown phases.
- **PEP 695 `type` statements** require Python 3.12+; project requires 3.14+ so OK. But mixing `type` statements (claude_payloads.py) with `Callable[..., None]` aliases (imports.py) is stylistically inconsistent.
- **`requirements.txt` mentions CVEs in 2026** (e.g. CVE-2026-34450) ŌĆö these are forward-dated, suggesting the project is intended for Python 3.14 / future releases. Verify the CVE numbers are real.
- **Verify discord.py 2.7.1 API usage**: `BotMissingPermissions.missing_permissions` was renamed (used to be `missing_perms` in older versions). Per discord.py changelog, `missing_permissions` was added in 2.0+ and is correct.
- **`ai_cog.py` has 1630+ lines** (per earlier grep) ŌĆö that's a large file. Future refactoring opportunity.
- **No structured concurrency**: `asyncio.create_task` is used with `add_done_callback` for some tasks (health loop) but not others (shutdown task).
