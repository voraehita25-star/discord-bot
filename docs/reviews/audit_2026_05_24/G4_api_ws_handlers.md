## Notes / non-issues confirmed during review

- **Document size cap before decode**: `dashboard_chat_claude.py:564-565` (PDF) and `:585-586` (text)
  estimate decoded size from base64 length and reject BEFORE `b64decode`, so the WS `max_frame`
  (~43 MB) is only an outer envelope; per-doc 32 MB cap is enforced pre-parse. CLI path mirrors this
  (`dashboard_chat_claude_cli.py:679/691`). Good.
- **`_on_endpoint_changed` broadcast** now uses `asyncio.gather` with per-client 2s timeout +
  dead-handle pruning (resolves prior MEDIUM about 20×2s serial blocking).
- **Failover client leak on shutdown**: `ws_dashboard.stop()` closes the SDK client + Gemini client;
  the failover-managed clients still aren't bulk-closed via a `close_all()` from `stop()`, but
  `_graceful_close` handles per-switch closure and `_pending_close_tasks` holds strong refs.
  Prior MEDIUM (no `close_all`) is PARTIALLY addressed; a full `APIFailoverManager.close_all()`
  on shutdown would still be cleaner. Not re-raised as a finding (low impact, single-process bot).
- **Background-task lifecycle**: tasks are tracked in `_background_tasks` + `_client_tasks`,
  cancelled on disconnect with bounded `wait_for`, and uncaught exceptions are surfaced via
  `_on_task_done`. Solid.
- **`hmac`, origin/host delimiter checks, bidi/zero-width title stripping** (`dashboard_handlers.py
  _BIDI_MARKS`) are all present and correct.
