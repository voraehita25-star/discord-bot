## Notes / cross-cutting

### Model-id correctness (explicitly requested)
- `claude-opus-4-7` = correct current GA id (WebSearch 2026-05-24). Migration 012 only changes DEFAULTs; existing rows keep their real model via `COALESCE(NULLIF(model,''), 'claude-opus-4-7')`. Defaults in `config.py:50` and `dashboard_config.py:82` agree. No mismatch.

### Migration system integrity (verified, mostly good)
- Atomic per-migration transactions with `schema_version` row in the same txn (`migrations.py:184-192`). Failures roll back and are fatal (`database.py:970-976`). Auto-backup (incl. WAL/SHM siblings, 5-backup retention) before applying pending migrations (`database.py:923-966`). This is solid.
- Idempotent re-apply guards (`DROP TABLE IF EXISTS *_new`) present in 003/007/010/011/012. Migration 011's comment notes it retro-added the guard that 003/007/010 had. Consistent.
- The runner's hand-rolled statement splitter (`migrations.py:116-182`) strips `--` and `/* */` comments before `sqlite3.complete_statement` so a `;` inside a comment won't split a statement — necessary and correct for these files.

### Path-with-spaces (`C:\BOT Discord`) — the recurring theme
- The one true BUG is `start.ps1:64` (HIGH). `_common.psm1` uses `Join-Path` throughout (good) and the `.bat` wrappers quote `%~dp0` (good). `run_tests.ps1:47` anchors with `Split-Path` (good). Build scripts use `Join-Path` + quoted `-o $Exe` (good). So spaces are handled everywhere EXCEPT the production `Start-Process -ArgumentList`.

### Download-and-run posture (`install_all.ps1`)
- Real effort was made (`Invoke-VerifiedDownload`: HTTPS-only, redirect cap, optional SHA). The gap is that **no call site supplies a hash**, so every elevated installer runs unverified. Three URLs are version-pinned (Git, Go, Node) and could be hash-pinned today with no downside. The Python alpha build (line 157) is a separate correctness smell.

### Shell-injection / `Invoke-Expression`
- No `Invoke-Expression`, no `iex`, no `& $unsanitizedVar` of attacker-controlled input found in any assigned PowerShell file. Native-command invocations use arrays/quoted paths. `%*` in the `.bat` files forwards to `-File` (params, not a command string) — not an injection vector.

### startup.json
- Schema/values sane. `max_restarts:50` is high (INFO). `colored_output`/`show_banner` booleans consumed correctly. The embedded fallback config in `_common.psm1:17-21` omits `check_dependencies` and `show_banner` — minor drift from the on-disk file (INFO at start.ps1:41).
