## Cross-cutting themes (MED/LOW)

- **datetime format inconsistency (HIGH + recurring):** mixing `datetime.now(timezone.utc).isoformat()` (T + offset + microseconds) into columns whose DEFAULT is SQLite `CURRENT_TIMESTAMP` (space-separated, no offset) breaks lexical `ORDER BY timestamp` on `ai_history` and corrupts the consolidator's partial-index sweep. `dashboard_messages` was correctly fixed to use `CURRENT_TIMESTAMP` everywhere (2208, 2216) — apply the same to `ai_history` writers.
- **Schema governance duplication (MED):** `init_schema` inline DDL vs. the migration runner are two competing sources of truth; migrations 009/014/015 are now pure presence-check SELECTs while the real DDL lives in `init_schema`. Drift risk is structural.
- **Export durability gaps (MED):** `KNOWN_TABLES` still missing `dashboard_document_memories` + `dashboard_conversation_tags` (silently excluded from JSON backups); `export_to_json` is unbounded-memory.
- **Multi-process safety (MED):** no `PRAGMA busy_timeout`; no UNIQUE on `ai_history(channel_id, local_id)`; `_write_lock` is process-local. Fine if single-process by design, but undocumented.
- **Pool accounting fragility (MED/LOW):** `_connection_count` mutated across ~7 paths; hard-resets to 0 in `close_pool*` can drive it negative; `_reinitialize_pool` rebinds the queue object (leaks in-flight conns into the abandoned queue). Semaphore handling is correct (BoundedSemaphore mutated in place).
- **Dead/declared-but-unused guard (LOW):** `_export_lock` (declared 120) is never acquired; the export scheduler's check-then-add relies on single-loop atomicity.

## SQL-injection posture

All user-data parameters are bound (`?`). Three dynamic-identifier f-string paths exist (`save_guild_settings`, `update_dashboard_conversation`, `increment_user_stat`, `export_to_json` table name) — each is whitelist-gated + identifier-regex-validated + bracket-quoted, and the export uses a **real** runtime check (not a strippable `assert`). No injection vector found. (G8 noted these as defense-in-depth; FIXED-VERIFIED.)
