Headline: the prior audit's most actionable concerns are largely FIXED-VERIFIED (entity upsert race → BEGIN IMMEDIATE; FAISS torn-save → save.uuid sidecar + length check; per-user dup race → per-user locks; `_ensure_index` lock-across-DB-scan → reconcile moved to thread + batch add; embedding dim validation; legacy pickle gated). Two NEW correctness issues dominate this round, both lifecycle/wiring rather than logic:

- **HIGH** — `SummaryArchiver` (memory_consolidator.py) is non-functional in production: its table `conversation_summaries` is created *only* by `init_schema()`, which is never called at startup, and the manual `!consolidate` command calls `consolidate_channel()` without initializing the schema → first real summary `INSERT` raises "no such table". The 6-hour background archival loop (`start_background_task`) is also never started → `ai_history` grows unbounded.
- **HIGH** — RAG long-term memory content is injected into the model prompt verbatim with no read-time sanitization (`logic.py:936-941`). Mitigated in practice because the only production writer (`remember` tool) scrubs at write time, but `add_memory()` itself performs no scrubbing and RAG memory is channel-scoped (one user poisons every user in the channel). Defense-in-depth gap with a real (if currently-bounded) injection/poisoning surface.

The remaining `MEMORY.md` prompt-injection concern from the prior audit — `long_term_memory.get_context_facts` injects `Fact.content` unsanitized — is STILL-PRESENT in the code but currently NOT WIRED into the live prompt (no caller in `logic.py`), so it's downgraded to LOW/latent.

3. **Possibly-dead Rust RAG path** (`rag_rust.py`): no production importer found; carries unresolved backend-divergence (#105) and load-validation (#108) risks for a code path that may never run.
4. **Per-window cost runaway** in the fact consolidator when extraction repeatedly fails (no decrement / no cooldown on the failure paths).
