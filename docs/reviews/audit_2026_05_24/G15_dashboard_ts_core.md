## Notes / cross-cutting

- **C3 (CRITICAL) is the headline:** the `package.json` bump to `dompurify ^3.4.2` did **not** update the runtime `ui/vendor/dompurify/purify.min.js`, which still banners `3.3.3`. Every `innerHTML = formatMessage(...)` sink in the assigned files inherits 3.3.3's CVEs. Fix = re-vendor + verify banner in CI.
- **H47 capabilities part is FIXED-VERIFIED:** `capabilities/default.json` no longer requests `core:default`; it now lists only `core:webview:deny-internal-toggle-devtools` + three dialog perms, with a rationale comment. The `connect-src` host was tightened (localhost dropped) but the **any-port** wildcard remains (HIGH, partial).
- **Positive deltas since prior audit:** textarea-decode XSS gadget removed (clipboard via `getAttribute`); `window.open` now `noopener,noreferrer`+`opener=null`+DOM build; sakura parallax + card-tilt listeners now use `AbortController`; localStorage `aiProvider` + settings now allowlist/range-validated.
- **Residual XSS posture:** with DOMPurify fixed and `withGlobalTauri:true` retained, the app is one same-origin-XSS away from full IPC. Per-field `escapeHtml` is applied consistently at the `innerHTML` template sites reviewed; the gaps are (a) the unescaped CSS selector (chat-manager 2378) and (b) untyped WS `e.detail` shapes feeding the failover renderers (app 1910-1977).
