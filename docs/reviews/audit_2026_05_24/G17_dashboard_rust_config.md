Threat model: single-user Windows desktop app; WebView2; bot + dashboard run as the same local user. No remote attack surface in the Rust layer itself (the WS server it talks to lives in the Python bot, audited in G3). Primary in-scope risks: (a) a same-origin XSS in the WebView reaching the 26 IPC commands, (b) local privilege/PATH issues in process spawning, (c) process-lifecycle correctness (orphans/zombies/races), (d) dependency CVEs.

Headline: the prior audit's two in-scope HIGHs (H47: `core:default` + wide `connect-src`) are **FIXED-VERIFIED**. The recent commit ae654ea's `start()` rework and `refresh_processes_with_cmd` helper are correct and well-documented. No CRITICAL/HIGH remain in the assigned Rust/config files. Remaining items are robustness, dependency hygiene (Linux-only transitive advisories that don't ship), and minor races.
