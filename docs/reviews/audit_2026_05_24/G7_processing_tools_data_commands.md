## Cross-cutting notes
1. **Input-side safety is not wired.** The single most important finding: `validate_input_for_channel`, `InputGuardrails.validate`, input `redact_secrets`, and control-char stripping are implemented and unit-tested but have **no production caller**. Output validation (`validate_response_for_channel`) IS wired (logic.py:1438). Net effect: prompt-injection detection, jailbreak scoring, and pre-storage secret redaction described in the module docstrings do not run on real user messages.
2. **Tool execution is half-dead.** `tool_definitions.py` (Gemini schemas) and `execute_tool_call` are not in the Claude turn loop (logic.py:155-165 documents this deliberately). `execute_tool_call` is still reachable from CLI/tests/dev-probes, so its `args=None` crash and error-echo issues remain live for those. The server-command path (`COMMAND_HANDLERS` via `PATTERN_SERVER_COMMAND`) appears to be the actually-wired mechanism — confirm whether `PATTERN_SERVER_COMMAND` is matched against model output in the response post-process (it is defined but I did not see a `.finditer`/`.search` consumer in the lines reviewed; flagged for verification by the core/logic group).
3. **Prior HIGHs in this scope are largely fixed.** H14 (repetition gate) FIXED-VERIFIED (runtime: "yes yes yes yes yes" now flagged). H17 (`MAX_DISCORD_LENGTH` alias) FIXED-VERIFIED (constants.py:88). H18 (roleplay `SERVER_CHARACTER_NAMES` = image paths) FIXED-VERIFIED (example.py:121-125 matches `media_processor.load_character_image:410`). H12 (remember authz) PARTIALLY-FIXED (admin-gated + attributed, but still channel-scoped, admin-not-owner, no rate limit).
4. **server_commands.py authz is solid.** Per-command `manage_channels`/`manage_roles`/`manage_messages`/`manage_guild` checks, role-hierarchy pre-validation (bot-vs-role AND bot-vs-target-member), `_SAFE_PERMISSIONS` allowlist + `_DANGEROUS_PERMISSIONS` denylist (administrator/manage_*/kick/ban/moderate excluded; `manage_threads` correctly promoted to dangerous), code-block escaping in `send_long_message`, and `_fmt_http_error` to avoid leaking raw API bodies. No kick/ban tools exist (good). This file is in good shape.
5. **No ReDoS** in the reviewed regex tables — adversarial 50K-100K inputs against the heaviest guardrails/injection patterns all completed <2 ms.
6. **No committed secrets/IDs** in the example data files — all IDs come from env (`_safe_int_env`), example persona text is generic placeholder.
