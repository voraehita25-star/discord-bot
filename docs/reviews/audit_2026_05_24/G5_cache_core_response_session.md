## Cross-cutting notes

1. **The cache `TokenTracker` is largely inert in production (HIGH).** `record_usage` and `start_cleanup_task` are never called outside tests; only `init_from_db` (startup) and `get_global_stats` (debug cmd) touch the global. Quota enforcement via `check_limits`, the batched persist loop, the cleanup loop, and the shutdown flush are all effectively dead. This is the cache-side of prior H33. Decide which of the two TokenTrackers is authoritative and either fully wire or delete this one. All the careful locking/batching fixes here are correct but currently exercise no production path.

2. **L2 SQLite connection is never closed (HIGH).** `flush_l2_pending` is now wired (good, H8 fixed) but `L2SqliteCache.close()` has no caller. Add it to `cog_unload`, ideally with a WAL checkpoint, so the last commits are durable on a clean shutdown.

3. **Analytics locking is now correct for iteration races (H9 fixed), but introduces a cross-thread blocking nuance (HIGH).** Holding `_sync_stats_lock` (a threading.Lock) from the event-loop thread means a sync reader on another OS thread can stall the loop. Keep all sync lock holds trivially short (percentiles already snapshot-then-release; summary/accuracy should too) or route hot reads through the async variant.

4. **Pricing & token estimation improved but not authoritative (MED).** Provider misclassification (H11) is fixed and the Thai CHARS_PER_TOKEN default (H10) is corrected + env-overridable. Both remain heuristics with hard-coded/undated rates and a single global char divisor; for anything user-visible (cost, quota) use the SDK token counter. NEEDS VERIFICATION of current Anthropic Opus-4.7 1M-context tier pricing.

5. **Prompt-injection surface in context_builder persists (MED).** Only URL content is wrapped (with a spoofable delimiter); RAG/entity/state are concatenated verbatim under trusted `## ` headers. Combined with the still-open `remember`-tool write path (prior G5 H12), a user can persist an injection that resurfaces in others' prompts.

6. **Message-queue lock lifecycle still fragile (MED).** Eviction-of-locked-channel corruption is FIXED (refuses when all locked). But ownership is untracked, `release_lock` pops `_lock_times` before its owner-guard, and stale locks are only logged, never force-released — a crashed processor bricks a channel until restart.

7. **Truncation invisibility (MED).** `response_sender.split_content` correctly preserves markdown across the 20-chunk cap (FIXED), but silently drops overflow with no `truncated` flag on `SendResult`, so callers can't react.
