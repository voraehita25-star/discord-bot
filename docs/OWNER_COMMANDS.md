## 💬 DM Chat - แชทส่วนตัว

แชทกับ AI ผ่าน Direct Message ส่วนตัวกับบอท **เฉพาะ Owner เท่านั้น**

### วิธีใช้

1. เปิด DM กับบอท
2. พิมพ์ข้อความอะไรก็ได้ (ไม่ต้องใช้คำสั่ง)
3. บอทจะตอบกลับโดยอัตโนมัติ

### ตัวอย่าง

```text
คุณ: สวัสดี
บอท: สวัสดีครับ! มีอะไรให้ช่วยไหม?

คุณ: อธิบายเรื่อง Python ให้หน่อย
บอท: Python เป็นภาษาโปรแกรมที่...
```

### คุณสมบัติ

- ✅ **แชทโดยไม่ต้องใช้คำสั่ง** - พิมพ์ข้อความได้เลย
- ✅ **บุคลิก Faust** - ยังคงเป็น Faust แต่ไม่มี roleplay actions
- ✅ **แชทแบบปกติ** - ไม่มี `>` actions หรือ *descriptions*
- ✅ **รู้สถานะ Voice** - Faust รู้ว่ากำลังอยู่ในห้องเสียงไหน กับใครบ้าง
- ✅ **มีความจำแยก** - ประวัติแชท DM แยกจาก channel อื่น
- ✅ **รองรับรูปภาพ** - ส่งรูปได้เหมือน channel ปกติ
- ✅ **ใช้คำสั่งได้** - `!rst`, `!thinking` ฯลฯ ทำงานได้ใน DM

### Voice Channel Awareness

Faust ใน DM สามารถรู้ได้ว่า:

- กำลังเชื่อมต่อกับห้องเสียงไหนบ้าง
- อยู่ใน Server และ Channel ชื่ออะไร
- กำลังเล่นเพลงอยู่หรือไม่
- **ชื่อเพลงที่กำลังเล่นอยู่**
- มีใครอยู่ในห้องเสียงด้วยบ้าง

```text
คุณ: ตอนนี้เธออยู่ไหน?

Faust: Faust กำลังเชื่อมต่อกับห้องเสียง "General" ใน Server "My Discord" 
อยู่ ในห้องมี Alice และ Bob อยู่ด้วย ขณะนี้ Faust กำลังเล่นเพลง 
"Never Gonna Give You Up" อยู่

คุณ: เล่นเพลงอะไรอยู่?

Faust: Faust กำลังเล่นเพลง "Bohemian Rhapsody" อยู่ในห้อง "Music Room"
มี 3 คนกำลังฟังอยู่ด้วย
```

### Chat History Access

Faust ใน DM สามารถเข้าถึงประวัติแชทของ **ทุก Channel** ได้:

#### ข้อมูลที่ Faust เห็น

```text
[Chat History Access]
📚 รายการ Channel ที่มีประวัติแชท AI:
• 1452373830755549277 (My Server/#general): 150 ข้อความ
• 1452626544307605604 (RP Server/#roleplay): 500 ข้อความ
• 1459575360357142629 (DM): 50 ข้อความ

💡 พูดถึง channel ID เพื่อขอดูประวัติแชท เช่น 'ดูประวัติ 1234567890'
```

#### วิธีขอดูประวัติแชท

เพียงพูดถึง Channel ID พร้อมคำขอ:

```text
คุณ: ดูประวัติ 1452626544307605604

Faust: Faust ได้ตรวจสอบประวัติแชทของ channel ดังกล่าวแล้ว 
พบการสนทนาเกี่ยวกับ... [สรุปเนื้อหา]

คุณ: สรุปบทสนทนาใน channel 1452373830755549277 ให้หน่อย

Faust: จากการวิเคราะห์ประวัติ 150 ข้อความล่าสุดในช่อง #general...
```

#### Keywords ที่ระบบรองรับ

- ประวัติ, history, ดู, อ่าน, ข้อความ, แชท, chat
- memory, ความจำ, channel, ช่อง, log, ดึง, โชว์, show

#### สิ่งที่ Faust ทำได้

| ความสามารถ | ตัวอย่าง |
| --------- | -------- |
| สรุปบทสนทนา | "สรุปให้หน่อยว่าคุยอะไรกัน" |
| ค้นหาเนื้อหา | "มีใครพูดถึงเรื่อง X ไหม" |
| วิเคราะห์ | "บทสนทนานี้มีโทนยังไง" |
| อ้างอิง | "ตอนนั้น AI ตอบว่าอะไร" |

### ความแตกต่างจาก Server Chat

| Feature | Server (Roleplay) | DM (Casual) |
| ------- | ----------------- | ----------- |
| บุคลิก | Faust | Faust |
| รูปแบบ | Roleplay + Actions | แชทปกติ |

| `*descriptions*` | ✅ มี | ❌ ไม่มี |
| Third-person | ✅ "Faust..." | ✅ "Faust..." |

### ตัวอย่างการตอบใน DM

```text
คุณ: วันนี้อากาศดีมากเลย

Faust: Faust ได้ตรวจสอบข้อมูลสภาพอากาศแล้ว อุณหภูมิอยู่ในช่วงที่

ประโยชน์
```

### หมายเหตุ

- ❌ ผู้ใช้อื่นที่ DM บอทจะไม่ได้รับการตอบกลับ
- ✅ Owner สามารถใช้งานได้ทันทีโดยไม่ต้องตั้งค่าเพิ่ม

## 🔗 Link Memory - คัดลอกความจำ

คัดลอกประวัติแชท AI จาก channel หนึ่งไปยัง channel ปัจจุบัน **โดยไม่ลบต้นทาง**

### คำสั่ง Link Memory

```text
!link_memory <channel_id>
!lm <channel_id>
!linkmem <channel_id>
```

### พารามิเตอร์ Link Memory

| พารามิเตอร์ | คำอธิบาย |
| ------------ | -------- |
| `<channel_id>` | ID ของ channel ต้นทางที่ต้องการคัดลอกความจำมา |
| `list` | แสดงรายการ channel ทั้งหมดที่มีความจำ AI |

### ตัวอย่าง Link Memory

```bash
# ดูรายการ channel ที่มีความจำ
!link_memory list
!lm list

# คัดลอกความจำจาก channel ID 1452428329146777822
!link_memory 1452428329146777822
!lm 1452428329146777822
```

### ขั้นตอน Link Memory

1. ระบบจะแสดงข้อมูลยืนยัน (source → target)
2. ตอบ `yes` เพื่อยืนยัน หรือ `no` เพื่อยกเลิก
3. ระบบจะคัดลอกประวัติทั้งหมดจาก channel ต้นทาง
4. Chat session จะถูก reload เพื่อใช้ความจำใหม่

### หมายเหตุ Link Memory

- ✅ **ต้นทางยังคงอยู่** - เป็นการคัดลอก ไม่ใช่ย้าย
- ✅ ประวัติเดิมของ channel ปลายทางจะยังคงอยู่ (เพิ่มต่อท้าย)
- ✅ AI จะเห็นความจำทั้งหมดรวมกัน

## 🚚 Move Memory - ย้ายความจำ

ย้ายประวัติแชท AI จาก channel หนึ่งไปยัง channel ปัจจุบัน **และลบต้นทาง**

### คำสั่ง Move Memory

```text
!move_memory <channel_id>
!mm <channel_id>
!movemem <channel_id>
```

### พารามิเตอร์ Move Memory

| พารามิเตอร์ | คำอธิบาย |
| ------------ | -------- |
| `<channel_id>` | ID ของ channel ต้นทางที่ต้องการย้ายความจำมา |
| `list` | แสดงรายการ channel ทั้งหมดที่มีความจำ AI |

### ตัวอย่าง Move Memory

```bash
# ดูรายการ channel ที่มีความจำ
!move_memory list
!mm list

# ย้ายความจำจาก channel ID 1452428329146777822
!move_memory 1452428329146777822
!mm 1452428329146777822
```

### ขั้นตอน Move Memory

1. ระบบจะแสดงข้อมูลยืนยันพร้อม **คำเตือนสีแดง**
2. ตอบ `yes` เพื่อยืนยัน หรือ `no` เพื่อยกเลิก
3. ระบบจะคัดลอกประวัติทั้งหมดจาก channel ต้นทาง
4. ระบบจะ **ลบประวัติจาก channel ต้นทาง**
5. Chat session จะถูก reload เพื่อใช้ความจำใหม่

### ⚠️ คำเตือน

- ❌ **ต้นทางจะถูกลบ** - ไม่สามารถกู้คืนได้!
- ✅ ประวัติเดิมของ channel ปลายทางจะยังคงอยู่ (เพิ่มต่อท้าย)
- ⚠️ ใช้ด้วยความระมัดระวัง

## 💡 ตัวอย่างการใช้งาน

### Scenario 1: รวมความจำจากหลาย Channel

```bash
# 1. ดูรายการ channel ที่มีความจำ
!lm list

# 2. คัดลอกความจำจาก channel A มายัง channel ปัจจุบัน
!lm 1234567890123456789

# 3. คัดลอกความจำจาก channel B มายัง channel ปัจจุบัน
!lm 9876543210987654321

# ตอนนี้ channel ปัจจุบันมีความจำจากทั้ง A และ B
```

### Scenario 2: ย้าย Channel (ปิด Channel เก่า)

```bash
# 1. ดูรายการ channel ที่มีความจำ
!mm list

# 2. ย้ายความจำจาก channel เก่ามายัง channel ใหม่
!mm 1234567890123456789

# Channel เก่าจะไม่มีความจำแล้ว (ถูกลบ)
```

### Scenario 3: Backup ความจำก่อนรีเซ็ต

```bash
# 1. ไปที่ channel สำรอง
# 2. คัดลอกความจำจาก channel หลักมา
!lm 1234567890123456789

# 3. กลับไป channel หลัก
# 4. รีเซ็ตความจำ
!rst

# ความจำยังอยู่ใน channel สำรอง สามารถ link กลับได้
```
