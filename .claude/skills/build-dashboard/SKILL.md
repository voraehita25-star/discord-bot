# Build & deploy the native dashboard (Tauri 2 + TypeScript)

The frontend is EMBEDDED in the binary (`tauri.conf.json` → `frontendDist: "./ui"`),
so any `src-ts/` change needs the cargo rebuild (which re-embeds `ui/`) to take effect —
a `tsc`-only build is not enough for the running app.

## Steps
1. **PATH + check it isn't running** (the copy step file-locks if the exe is open):
   ```powershell
   Get-Process -ErrorAction SilentlyContinue | Where-Object { $_.Path -like '*native_dashboard\target\release*' }
   ```
   If running and you need the new build live, the user launches
   `native_dashboard\target\release\디스코드 봇 대시보드.exe` — close it first (Get-Process name won't match "dashboard"; match by path or the Korean name), then build.

2. **Build (needs the MSVC env for the Rust/Tauri link):**
   ```
   cmd /c "set PATH=%USERPROFILE%\.local\node;%USERPROFILE%\.cargo\bin;%PATH% && call \"C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools\VC\Auxiliary\Build\vcvars64.bat\" && cd /d \"C:\BOT Discord\native_dashboard\" && npm.cmd run build:release"
   ```
   `build:release` = `tsc` → `cargo build --release` → copies `bot-dashboard.exe` to 5 destinations under 3 names (incl. `디스코드 봇 대시보드.exe`): 2 in `target/release/` and 3 in the repo root.

3. **Verify**: the exes in `native_dashboard/target/release/*.exe` have a fresh timestamp (~16.6 MB).

## Korean desktop shortcut (if asked)
Run `native_dashboard/scripts/create_desktop_shortcut.py` with `PYTHONUTF8=1` set (else `UnicodeEncodeError` on the Korean name). NEVER create/read the Korean `.lnk` via `WScript.Shell` from PowerShell (COM mangles the Korean path) — use python `pythoncom`/IShellLink (pywin32 is in the venv).

Tracked compiled files under `native_dashboard/ui/` are regenerated by the `tsc` step; `ui/app.js` is gitignored.
